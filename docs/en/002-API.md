# API

The API is options-first and designed for migration from `act-js` without copying its unsafe behaviors.

## GitHub Facade

```ts
import { ActRunner, withMockGithub } from "pretend-act/github";
```

`ActRunner` provides mutable setters for migration ergonomics and options-first run methods:

- `setEnv(key, value)`
- `setInput(key, value)`
- `setSecret(key, value)`
- `setVar(key, value)`
- `setCustomContainerOpts(value)`
- `validateWorkflow(options?)`
- `dryRunWorkflow(event, options?)`
- `runEvent(event, options?)`
- `runJob(jobId, options?)`

`withMockGithub(options, callback)` creates a staged temporary repository, passes it to the callback, and cleans it up when the callback finishes.

## Git Registry Checkout

```ts
import { createCheckoutMockStep, createGitRegistry, startGitHttpTransport } from "pretend-act/git-registry";
```

`createGitRegistry(options)` creates a local Git registry for checkout simulation. In auto mode, dirty workspaces become filtered snapshot commits; explicit `sourceSha`, tags, and clean `sourceRef` branches publish the selected commit directly. The returned object includes `remoteUrl`, `transport`, `fetchRef`, `checkoutSha`, `mode`, and `stop()`. The default transport returns a local `file:` URL; set `transport: "http"` when the runner needs to fetch over HTTP. `startGitHttpTransport(options)` can also expose an existing registry directory.

`createCheckoutMockStep(server)` returns a shell step that can replace `actions/checkout` through `mockSteps`. `pretend-act/github-registry` and `pretend-act/github` re-export these helpers for GitHub workflow callers.

## Workflow Overlay

```ts
mockSteps: {
	"release-plan": [
		{
			uses: "actions/checkout@v6",
			mockWith: { if: "${{ false }}" },
		},
	],
}
```

Steps can be selected by `id`, `name`, `uses`, `run`, `index`, `before`, or `after`. The overlay is written only to sandbox workflow files.

## Result

```ts
type RunResult = {
	command: CommandSpec;
	status: "success" | "failure" | "cancelled" | "unknown";
	success: boolean;
	exitCode: number | null;
	stdout: string;
	stderr: string;
	rawLog: string;
};
```

Raw log and exit code are the reliable source. Step-level summaries are conservative and may be `unknown`.

## Registry Helpers

`pretend-act/npm-registry` provides configuration fragments and Verdaccio lifecycle helpers. `pretend-act/crates-registry` provides Cargo config plus a sparse registry server with core Web API support. `pretend-act/docker-registry` starts and stops an official OCI Distribution registry container. `pretend-act/github` exposes `createRemoteMockEnvironment` to orchestrate these services and deny real publishing endpoints by default.
