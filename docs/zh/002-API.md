# API

API 使用 options-first 风格，同时照顾从 `act-js` 迁移的手感，但不会复制旧实现的不安全行为。

## GitHub Facade

```ts
import { ActRunner, withMockGithub } from "pretend-act/github";
```

`ActRunner` 提供便于迁移的 setter，以及 options-first 的运行方法：

- `setEnv(key, value)`
- `setInput(key, value)`
- `setSecret(key, value)`
- `setVar(key, value)`
- `setCustomContainerOpts(value)`
- `validateWorkflow(options?)`
- `dryRunWorkflow(event, options?)`
- `runEvent(event, options?)`
- `runJob(jobId, options?)`

`withMockGithub(options, callback)` 创建 staged temporary repository，传给 callback，并在 callback 完成后清理。

## Git Registry Checkout

```ts
import { createCheckoutMockStep, createGitRegistry, startGitHttpTransport } from "pretend-act/git-registry";
```

`createGitRegistry(options)` 会创建一个用于 checkout simulation 的本地 Git registry。auto mode 下，脏工作区会变成 filtered snapshot commit；显式 `sourceSha`、tag 和 clean `sourceRef` branch 会直接发布选定 commit。返回对象包含 `remoteUrl`、`transport`、`fetchRef`、`checkoutSha`、`mode` 和 `stop()`。默认 transport 返回本地 `file:` URL；当 runner 需要通过 HTTP fetch 时设置 `transport: "http"`。`startGitHttpTransport(options)` 也可以暴露已有 registry directory。

`createCheckoutMockStep(server)` 返回一个 shell step，可通过 `mockSteps` 替换 `actions/checkout`。`pretend-act/github-registry` 和 `pretend-act/github` 会为 GitHub workflow 调用方 re-export 这些 helper。

## Workflow Overlay

```ts
mockSteps: {
	"release-plan": [
		{
			uses: "actions/checkout@v6",
			mockWith: { if: "${{ false }}" },
		},
	],
}
```

Step 可以按 `id`、`name`、`uses`、`run`、`index`、`before` 或 `after` 匹配。Overlay 只写入 sandbox workflow 文件。

## Result

```ts
type RunResult = {
	command: CommandSpec;
	status: "success" | "failure" | "cancelled" | "unknown";
	success: boolean;
	exitCode: number | null;
	stdout: string;
	stderr: string;
	rawLog: string;
};
```

Raw log 和 exit code 是可靠来源。Step 级 summary 保守处理，无法可靠推导时会是 `unknown`。

## Registry Helpers

`pretend-act/npm-registry` 提供配置片段和 Verdaccio lifecycle helper。`pretend-act/crates-registry` 提供 Cargo config，以及具备核心 Web API 的 sparse registry server。`pretend-act/docker-registry` 负责启动和停止官方 OCI Distribution registry container。`pretend-act/github` 暴露 `createRemoteMockEnvironment` 编排这些服务，并默认 deny 真实发布端点。
